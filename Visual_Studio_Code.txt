===Neues Projekt/Workspace erstellen===
---> wenn man ein bestehendes Projekt zum ersten Mal auf GitHUB stellen möchte
+ wks ordner auf hdd erstellen
+ mit cmd in wks ordner rein: code . ODER VS Code starten 
+ den wks ordner "add folder to workspace" über FILE
+ "Save Workspace As" AU?ERHALB des wks ordner
+ index.html, .gitignore, README.md erstellen

+ auf GitHUB neue repo erstellen <url>

>git init
>git config --global user.name <name> --- Define the author name to be used for all commits by the current user
>git config --global user.email <email> --- Define the author email to be used for all commits by the current user
git add README.md
git commit -m "first commit"
git branch -M main (im wks ordner:>git init -b main  (neue default branches heißen main und nicht master. Nachschauen wie es heißt bevor unnötig umbenennen)
git remote add origin <url>
git push -u origin main oder >git push --force -u origin main (falls oben schon ein README.dm ist)

===In neues Projekt von GitHub pullen===
>git --version
>git init
>git status (check if master or main in online repo)
>git config --global user.name <name> --- Define the author name to be used for all commits by the current user
>git config --global user.email <email> --- Define the author email to be used for all commits by the current user
>git clone https://github.com/geronimoooooooo/proj.git
